# Intent: 启动 Lainclaw 文档库（轻量级版本）

> **状态**: DRAFT
> **创建日期**: 2026-02-21
> **对应 Issue**: #N/A

## 1. 背景与动机 (Context & Motivation)

* **现状**: 现有文档库仍带有 `repo-a/repo-b`、`TODO`、模板占位符等旧内容，且目录命名/定位与目标产品（`lainclaw`）不一致。
* **问题**: 当前文档无法直接支撑新项目立项。
* **价值**: 用统一且可持续的文档骨架取代占位符内容，完成从 OpenClaw 语境到 Lainclaw 语境的迁移。

## 2. 核心目标 (Goals)

1. [ ] 形成一套可执行的 Lainclaw 文档初始化 WIP 草案。
2. [ ] 用明确项目语义替换 `repo-a/repo-b`、`TODO` 与模板占位符。
3. [ ] 完成文档库关键入口文件的首次“非空、可读、可执行”初始化。

## 3. 非目标 (Non-Goals / Out of Scope)

* [ ] 不涉及运行时代码与业务逻辑改造。
* [ ] 不进行 CI/CD 或权限体系设计。
* [ ] 不重构现有源码目录结构。

## 4. 用户故事 (User Stories)

* **As a** 项目负责人
* **I want to** 快速启动 Lainclaw 的文档体系
* **So that** 之后的需求、架构、任务都能按文档驱动流程进行

## 5. 风险评估 (Risks)

* 可能把模板约束替换得过重，导致未来需求表达空间受限。
* 可能出现文档与实际代码边界（`docmap.yaml`）不一致；需通过验收确认对齐。
