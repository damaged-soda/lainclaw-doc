# Intent: Lainclaw Heartbeat MVP（自然语言提醒）

> **状态**: DRAFT
> **创建日期**: 2026-02-22
> **对应 Issue**: TBD

## 1. 现状与背景 (Context & Motivation)

* **现状**: `lainclaw` 已具备 `ask` 与 Feishu 网关能力，但还没有内建“按间隔评估提醒项并主动发起告警/消息”的闭环机制。
* **问题**: 用户希望先用较轻量能力快速落地“提醒”体验，避免先做复杂日历/结构化解析。
* **价值**: 用同一套模型交互能力 (`runAsk`) 做自然语言规则评估，先得到可用的闭环路径（可人工验证）后再逐步扩展。

## 2. 核心目标 (Goals)

1. [ ] 提供最小可用的心跳规则存储能力（文件化持久化）。
2. [ ] 提供 `lainclaw heartbeat` 命令用于规则增删改查与手工触发。
3. [ ] 在 `gateway --channel feishu` 运行时支持定时触发心跳任务，并把触发内容回发到配置的 Feishu 接收者。
4. [ ] 规则以自然语言文本为主，不做固定结构解析；支持可选 `tool` 调用路径。
5. [ ] 给出可验证验收标准（无固定格式/最小化依赖）。

## 3. 非目标 (Non-Goals / Out of Scope)

* 不在本次实现中接入外部日历、Cron 表达式解析或复杂结构化提醒 DSL。
* 不改动 `ask` 与现网工具注册核心机制；只做最小闭环调用。
* 不新增跨仓库持久化协议或数据库，沿用本地 JSON 文件。

## 4. 用户故事 (User Stories)

* **As a** 个人 AI 助手使用者
* **I want to** 记录“自然语言提醒项”并让网关按间隔评估
* **So that** 到时自动发消息给我，而不是每次手工触发。
