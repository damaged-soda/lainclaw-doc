# Intent: 去掉飞书/统一入口默认 openai-codex 前缀响应

## 背景
当前 `openai-codex` 适配器返回的文本默认会带有形如 `[openai-codex:<profileId>]` 的前缀，飞书消息会直接把这段调试信息透传给用户，造成体验噪音。

## 目标
- 默认输出应只保留模型正文，不带来源标识前缀。
- 保留必要时可开启的调试行为：仅在显式开启开关后，附带 provider/profile 调试前缀，便于问题排查。

## 非目标
- 不改变 openai-codex 接口调用主流程、鉴权、工具调用链路与会话上下文行为。
- 不调整飞书网关的配对、权限、超时或持久化策略。
- 不更改 `runAsk` 的返回结构字段语义（`result`、`sessionKey` 等）。

## 验收标准
- `runCodexAdapter` 在未开启调试开关时返回纯文本内容。
- 开启调试开关时，返回格式按预期带上 `[openai-codex:<profileId>] ` 前缀。
- 飞书网关不再默认向用户回传上述前缀。
